## Technical decisions

- **2026-07-12 — No npm workspaces.** Each package (`api/`, `mobile/`, `web/`, `data/`) manages its own dependencies. Expo's Metro bundler is friction-prone with hoisted node_modules; workspaces buy nothing at this scale.
- **2026-07-12 — Local Node is v24** (satisfies `engines >=22`). Railway deploy will pin Node 22 per the architecture doc.
- **2026-07-12 — GitHub repo `danzoy-ship-it/tapowner` pre-existed as PUBLIC.** Flagged to Frederick: consider making it private (repo Settings → Change visibility) — the build doc contains the full pricing model and margin math.
- **2026-07-12 — `web/` and `data/` are README placeholders** until their phases (5 and 1 respectively) — Phase 0 scope is API hello-world + Expo blank app only.
- **2026-07-12 — Railway deploys are GitHub-sourced, not CLI-uploaded.** `railway up` from this Windows machine dies instantly on the Metal builder with no logs (twice with Railpack, once with Dockerfile). Connecting the service source to the GitHub repo builds fine — and gives push-to-deploy on `main`. Root `Dockerfile` builds `api/` (context = repo root); when `web/` ships it gets its own service with a root-directory setting.
- **2026-07-12 — db service: `PGDATA=/var/lib/postgresql/data/pgdata`.** Railway volumes mount with `lost+found`, which makes initdb refuse the mount root; pointing PGDATA one level down is the standard fix.
- **2026-07-12 — StratMap common schema does not carry living area, beds/baths, stories, pool/garage indicator, or sale price/date for ANY county** — confirmed against the real 2025 Bexar file (709,541 features): these columns simply don't exist in the .dbf. TxGIO's own description confirms why: "StratMap Land Parcels are intended to be used as basemap and cartographic data and are not for detailed analysis." Per TAPOWNER_BUILD.md §6's own footnote ("later we refresh big-metro counties quarterly from CAD exports directly — documented future task, not v1"), v1 proceeds StratMap-only; those columns are null for every parcel until a direct-CAD-export source is ingested (not v1). Flagging because Phase 4's UI promises these details per-property — that promise can't be met for ANY property under the current v1 data source, not just per-county coverage gaps. Business/roadmap conversation, not a blocker for Phase 1 itself.
- **2026-07-12 — StratMap is public, unauthenticated, CC0-licensed, one zip per county** — found via the DataHub's S3 warehouse (`s3://tnris-data-warehouse/LCD/collection/stratmap-2025-land-parcels/items/{shp,fgdb}/`), 253 of 254 counties present. No TxGIO account/token needed. Harris County ships as two shapefiles (east/west split — its dbf would exceed size limits otherwise); loader handles multi-shapefile counties.
